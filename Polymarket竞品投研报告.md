      ·····                                                                          # Polymarket 数据工具竞品分析与产品规划

## 第一部分：需求分析

### 1.1 市场背景

Polymarket 在过去两年完成了从小众预测市场到主流交易平台的转变。2025 年全年交易量达到 215 亿美元，月活跃用户从 2024 年初的 4000 人增长到 2025 年底的 60 万人以上。2026 年 2 月单月交易量突破 70 亿美元，2 月 28 日创下单日 4.25 亿美元的历史记录。

这个增速带来了一个问题：平台官方只提供基础的交易界面，缺乏专业交易者需要的数据分析能力。于是第三方工具开始填补这个空白。

### 1.2 目标用户画像

根据现有工具的定位，Polymarket 交易者可以分为三类：

**信息型交易者**
- 依赖对事件本身的判断进行交易
- 需要：事件背景、新闻聚合、概率走势
- 痛点：信息分散，难以追踪事件进展与价格变化的关系

**技术型交易者**
- 用价格行为和技术指标辅助决策
- 需要：K线图、成交量、订单簿深度、技术指标
- 痛点：Polymarket 原生界面只有简单折线图，没有专业图表

**跟随型交易者**
- 相信某些"聪明钱"地址的判断
- 需要：鲸鱼追踪、地址标签、跟单功能
- 痛点：不知道谁是高胜率玩家，缺乏系统性的地址分析

### 1.3 用户痛点整理

从 Reddit、Twitter 和各工具的功能定位可以归纳出以下痛点：

| 痛点 | 具体表现 |
|------|---------|
| 信息不对称 | 内幕交易者提前布局，普通用户发现时价格已经反映消息 |
| 数据分散 | 看K线要用一个工具，看鲸鱼要用另一个，看新闻要去第三个地方 |
| 图表原始 | 官方界面只有折线图，没有K线、技术指标、成交量叠加 |
| 跟单盲目 | 知道有大单进场，但不知道这个地址历史表现如何，没有回测验证 |
| 缺乏提醒 | 需要持续盯盘，错过重要价格变动 |
| 中文有限 | 仅 Whales Market 支持中文，但功能不完整；缺乏中文内容和事件翻译 |
| 新闻关联弱 | 难以快速判断某条新闻会影响哪些市场 |

## 第二部分：竞品分析

按照产品形态，现有工具可以分为四类。下面逐一分析。

### 2.1 专业交易终端

这类产品试图复制 Bloomberg Terminal 或 TradingView 的体验。

**Verso（verso.trading）** ⭐ 重点关注

Y Combinator 背书的专业交易终端，定位是"预测市场的彭博终端"。支持 Kalshi 和 Polymarket 双平台。

核心功能：
- 实时追踪 15,000+ 预测市场合约
- AI 新闻引擎：用 LLM 嵌入和 GPT-5 将 30,000+ 新闻文章映射到相关合约，73% 准确率识别市场催化剂
- 高级市场筛选器：按类别、价格变动、交易量、到期时间等过滤
- 智能提醒系统：价格变动、价差变化、交易量飙升、新闻事件的 Web/移动推送
- 事件聚合：将重复标题合并为单一事件，减少信息噪音

做得好的地方：
- YC 背书，产品完成度高
- AI 新闻关联是核心差异化，将新闻自动关联到相关市场
- 跨平台支持（Kalshi + Polymarket）

不足：
- 目前仍在 V0 Beta 阶段
- 主要服务 Kalshi 用户，Polymarket 功能相对较弱
- 没有鲸鱼追踪和跟单功能

**Whales Prediction（whales.market/prediction）** ⭐ 重点关注

高性能交易终端聚合器，定位是"#1 Trading Terminal Aggregator"。**支持中文界面**。

核心功能：
- 多平台聚合：整合 Polymarket、Kalshi 等主要预测市场
- 高级筛选：按平台、概率、交易量、流动性、市场年龄、价差过滤
- Reefboard 多市场执行：同时管理最多 8 个订单簿
- 智能钱监控：鲸鱼持仓和实时活动信号层
- 统一仓位追踪：跨市场 P&L 监控和活动历史

做得好的地方：
- **唯一支持中文的专业终端**，直接威胁我们的差异化
- 聚合器定位，比单一平台工具更全面
- 执行效率高，宣称比分散交易提升 1-3% 执行效果
- 决策时间减少 10x

不足：
- 仍在早期 Beta 阶段
- 没有详细的地址分析和跟单功能
- 缺少 K 线图和技术指标

**PolyMoon（polymoon.io）**

定位是"Polymarket 的彭博终端"。Web 端提供 TradingView 风格的 K 线图，实时订单簿可视化，支持鲸鱼交易追踪。同时有 Telegram Bot 可以在手机上下单。宣称零交易费。

做得好的地方：
- 把图表、订单簿、鲸鱼追踪整合在一个界面
- 跨端体验（Web + Telegram）

不足：
- 界面信息密度偏低，专业交易者可能觉得不够用
- 没有详细的地址分析和历史胜率数据

**TradeFox（thetradefox.com）** ⭐ 重点关注

Alliance DAO 和 CMT Digital 背书的专业预测市场聚合器和 Prime Brokerage 平台，目前处于 Private Beta 阶段。

核心功能：
- 高级订单类型：市价单、限价单、TWAP（大额交易分拆）、追踪止损、自动止盈止损
- 智能市场发现：按截止时间、概率、流动性、交易量、关键词过滤数千个市场
- 多资产支持：接受 USDC、BTC、ETH、SOL 入金，自动转换为 Polygon 上的 USDC.e
- 100% 自托管：基于 Privy 的 SOC 2 合规基础设施，TEE 可信执行环境，无 KYC

费率结构：
- 阶梯费率：0.95%（Cub 级）到 0.75%（TradeFox Prime）
- 最高 0.25% 返现奖励

未来规划：
- 杠杆交易、Parlays（组合投注）、跟单交易、直播功能
- 扩展支持 Kalshi、Limitless、SxBet、Myriad

做得好的地方：
- 机构级订单功能（TWAP、追踪止损）是独特差异化
- 顶级 VC 背书（Alliance DAO + CMT Digital）
- 自托管 + 无 KYC，符合 Crypto Native 用户偏好

不足：
- 仍在 Private Beta，公开信息有限
- 有交易费用（0.75%-0.95%），不如 Bullpen 零手续费
- 无中文支持

**Based.one（app.based.one/predict）**

集成 Polymarket 的综合交易平台，提供交易、预测、消费一体化体验。

核心功能：
- Polymarket 集成：通过 Based Terminal 和移动 App 访问数百个预测市场
- Gold 奖励机制：预测活动可获得 Gold 奖励，单 Epoch 上限 200 万 Gold
- 低费率：买入时收取可变费率，公式为 `fee = round up (0.07 × C × P × (1 – P))`，卖出无费用

未来规划：
- Mobile Prediction
- BasedPals 市场

做得好的地方：
- 交易 + 预测 + 消费的一体化定位
- 奖励机制可能吸引用户
- 移动端友好

不足：
- 目前处于 Beta 阶段
- 部分地区因监管限制无法使用
- 功能相对基础，无高级分析

### 2.2 鲸鱼追踪 / 聪明钱分析

这是目前最热门的赛道，直接对标 GMGN 在 Meme 币领域的定位。

**insiders.bot** ⭐ 重点关注

老牌玩家，功能最完整的聪明钱分析平台之一。

核心功能：
- 市场信号：按 Finance/Crypto、Politics、Sports、Tech/Culture 分类的实时市场
- 钱包监控（WALLETS）：追踪特定钱包的交易活动
- 高级分析器（ANALYSER）：深度交易行为分析
- 跟单交易（COPY）：一键复制高胜率钱包的交易

做得好的地方：
- 功能闭环完整：分析 + 监控 + 跟单
- 概念差异化：用"行为感知"检测异常交易模式
- 品牌认知度高，老牌玩家

不足：
- 产品相对封闭，外部很难评估实际效果
- 没有公开的交易者排行榜或历史验证
- 无中文支持

**Bullpen.fi（app.bullpen.fi/prediction）** ⭐ 重点关注

定位是 Polymarket 的分析与交易层，核心是"显示 Polymarket 隐藏信号"。

核心功能：
- Live Trades 视图：实时显示鲸鱼大单、大额持仓建立
- 钱包追踪：发现并监控盈利钱包
- 性能指标：30天滚动 P&L、胜率、最大盈利
- Push 通知：追踪钱包交易时推送提醒
- Copy Trade：一键复制交易

交易特点：
- 零手续费（无 maker/taker/赎回费）
- 通过 Polymarket CLOB 执行，链下撮合 + 链上结算
- 支持 GTC/FOK/GTD 多种订单类型

做得好的地方：
- 工作流清晰：通知 → 查看 → 复制，三步完成
- 零手续费降低交易门槛
- 有博客内容做用户教育

不足：
- 没有专业 K 线图和技术指标
- 无中文支持
- 地址标签不够精细（只有胜率/P&L）

**Hashdive（hashdive.com）** ⭐ 重点关注

专业的 Polymarket 和 Kalshi 分析平台，功能完整度很高。

核心功能：
- Smart Scores：交易者一致性和盈利能力评分（-100 到 100），综合考虑已关闭和未关闭仓位
- 市场筛选器：按流动性、交易量、价格变动、鲸鱼活动、动量过滤
- 高级图表：K 线图 + RSI/MACD/均线等技术指标 + 鲸鱼交易量分解
- 钱包追踪：交易历史、P&L、持仓、性能指标（Pro 版最多 50 个地址）
- Bot/鲸鱼检测：识别高频交易和新钱包大额持仓等异常行为
- 内幕检测：标记可能受非公开信息影响的异常交易活动
- 流动性洞察：订单簿深度可视化

做得好的地方：
- Smart Scores 评分系统是差异化卖点
- 图表 + 鲸鱼分析整合，功能不割裂
- 同时支持 Polymarket 和 Kalshi

不足：
- 无跟单功能
- 无中文支持
- 定价信息不透明

**Predicts.guru（predicts.guru）**

定位是"Polymarket 钱包智能分析"。

核心功能：
- 钱包检查器：输入地址或用户名，查看完整交易统计、性能指标、活动历史
- 事件分析：交易量分布、概率趋势、结果分析
- 实时活动：市场实时交易和交易量数据
- 排行榜：按 24 小时实现 PnL 排名的顶级交易者

做得好的地方：
- 使用门槛低，输入地址即可分析
- 有排行榜帮助发现聪明钱

不足：
- 功能相对简单，主要是数据展示
- 无跟单功能
- 无技术图表

### 2.3 跨平台套利工具

这类产品专注于发现不同预测市场平台之间的价差套利机会。

**Matchr（matchr.xyz）**

定位是"跨平台预测市场"，核心功能是发现不同平台间的套利机会。

核心功能：
- Markets：浏览各平台市场
- Matches：发现同一事件在不同平台的价差
- Rewards：奖励机制
- Portfolio：跨平台持仓管理

做得好的地方：
- 套利发现是独特价值
- 跨平台整合视角

不足：
- 产品信息较少，功能完整度待验证
- 套利机会可能转瞬即逝，对延迟要求高

### 2.5 社区与 Discord Bot

这类产品将交易体验整合到社区中，降低使用门槛。

**Betmoar（betmoar.fun）** ⭐ 重点关注

Polymarket 官方合作的 Discord Bot，市场占有率超过 70%。

核心数据：
- 50M+ 美元交易量
- 100+ 合作 Discord 服务器
- 50,000+ 社区成员

核心功能：
- 高级市场视图：持仓 Delta、交易者分析、用户画像
- UMA 争议追踪：直观的争议市场仪表板，实时投票和大额未披露投票者追踪
- 集成新闻终端：AI 驱动的新闻聚合，自动关联到相关市场，TTS 语音提醒
- 四种发现视图：Movers（价格变动）、Bonds（高概率终局市场）、Disputes（UMA追踪）、Comments（社交活动）
- 零手续费交易：直接在 Discord 内下单

做得好的地方：
- **社区化交易**：在 Discord 内完成交易闭环，用户无需离开社区
- UMA 争议分析是独特功能，可在结算前发现机会
- AI 新闻 + TTS 确保不错过重要信息
- 官方合作背书，信任度高

不足：
- 依赖 Discord 平台，Web 体验相对较弱
- 功能分散在 Bot 命令中，学习成本较高
- 无中文支持

### 2.6 K线图表工具

纯图表工具，不做鲸鱼分析。

**UpDown Charts（updowncharts.com）**

专注 BTC/USDT 预测市场的 K 线图。500ms WebSocket 更新，5 个同步图表（不同时间周期），7000+ 历史数据点。

定价：20 美元终身买断，含 API 和 CSV 导出。

做得好的地方：
- 数据刷新最快（500ms）
- 一次性付费，用户心理负担小

不足：
- 只覆盖 BTC 相关市场，其他事件市场没有
- 功能单一，没有交易者分析

### 2.7 数据 API

面向开发者的基础设施。

**Polymarket 官方 API**

提供 REST 和 WebSocket 两种接入方式。REST 端点可以拿到预聚合的 OHLC K 线数据，支持 1 分钟到 1 天的多种周期。WebSocket 有 4 个频道：Market（订单簿、价格、交易）、User（订单状态）、Sports（比分）、RTDS（评论、加密货币价格）。

优势：官方数据源，免费

**PredictAPI（predictapi.dev）**

第三方数据服务商。12ms 平均延迟，支持套利检测，有 tick 级历史数据。

定价：免费版 100 次请求/天，Pro 版 29 美元/月（10000 次/天），Enterprise 版 99 美元/月（无限制 + 1 年历史数据）。

优势：比官方 API 更专业的封装，有增值功能（套利检测）

#### 竞争格局变化（2026 年 3 月更新）

**威胁评估：**

1. **Whales Market 已支持中文** ⚠️
   - 这是对我们"中文首发"差异化的直接威胁
   - 但其功能偏聚合器，缺少图表和跟单，仍有空间

2. **Verso 的 AI 新闻引擎很强**
   - GPT-5 驱动的新闻关联是我们规划的"事件时间线"的竞品
   - 但 Verso 主攻 Kalshi，Polymarket 不是其重点

3. **Bullpen 验证了跟单闭环**
   - 通知 → 查看 → 复制 的工作流已被验证
   - 但缺少图表，与我们不直接冲突

4. **Hashdive 功能最完整**
   - 图表 + 技术指标 + 鲸鱼追踪 + Smart Score
   - 是最接近我们定位的竞品，需要重点关注

5. **Betmoar 的社区化模式值得借鉴**
   - Discord 内交易闭环，UMA 争议追踪是独特功能
   - 可考虑做 Telegram 群组内的类似体验

#### 现有产品的共同问题

1. **功能割裂**：做图表的不做鲸鱼分析，做鲸鱼的不做图表
2. **缺乏整合**：用户需要同时开 3-4 个工具才能获得完整信息
3. **中文有限**：只有 Whales Market 支持中文，但功能不够完整
4. **移动体验差**：大部分是 Web/插件，移动端只能靠 Telegram Bot
5. **跟单盲目**：即使有跟单功能，也缺少回测验证机制

## 第三部分：产品规划

基于以上分析，我们的产品应该解决"数据分散"这个核心痛点，做一个整合型的 Polymarket 数据平台。

### 3.1 产品定位

**一句话定位**：Polymarket 版的 GMGN——聪明钱追踪 + 专业图表 + 实时提醒，中文优先。

**目标用户**：中文 Polymarket 交易者，尤其是从 Meme 币交易迁移过来、熟悉 GMGN 使用逻辑的用户。

### 3.2 核心功能规划

**第一阶段：数据基础（MVP）**

| 功能模块 | 具体内容 | 参考竞品 |
|---------|---------|---------|
| K线图表 | TradingView 风格，支持 1m/5m/15m/1h/4h/1d 周期，RSI/MACD/MA | Hashdive, PolyFire |
| 鲸鱼 Feed | 实时大单（$5000+）流，显示地址、金额、方向 | Bullpen, PolyTrack |
| 地址详情页 | 历史交易、30天 P&L、胜率、持仓分布 | Bullpen, 0xInsider |

### 3.3 借鉴 GMGN 的设计要点

GMGN 能在 Meme 币工具里脱颖而出，有几个值得学习的点：

**1. 地址标签体系**

GMGN 给地址打标签：Smart Money、Sniper、KOL、Paper Hand、Diamond Hand。用户一眼就能判断一个地址的"身份"。

我们可以针对 Polymarket 设计标签：
- 🎯 Sharp（高胜率）
- 🐋 Whale（大资金）
- ⚡ Scalper（短线高频）
- 📊 Market Maker（做市商）
- 🔮 Event Specialist（特定领域专家，如政治、体育）
- ⚠️ Insider Suspect（疑似内幕交易）

**2. 信息密度**

GMGN 的界面信息密度很高，一屏能展示大量数据。这和很多 Web3 产品"留白多、信息少"的风格不同，但专业交易者喜欢。

我们应该采用类似的高密度设计：
- 左侧：事件列表/热门市场
- 中间：K线图 + 成交明细
- 右侧：鲸鱼 Feed + 地址详情
- 底部：排行榜 Tab

**3. 操作路径短**

GMGN 做到了"看到 → 判断 → 下单"三步完成。我们也要尽量缩短路径：
- 在鲸鱼 Feed 里直接能点击跟单
- 地址详情页有"关注"和"复制交易"按钮
- 图表页面有快速下单入口

### 3.4 差异化策略

**1. 中文深度支持**（调整策略）

Whales Market 已支持中文，但其功能不够完整。我们的差异化不仅是"有中文"，而是：
- 中文事件名称翻译（如"Fed rate cut" → "美联储降息"）
- 中文新闻聚合和分析
- 中文社区和客服
- 符合中国用户习惯的 UI/UX

**2. 事件时间线叠加**

Polymarket 和 Meme 币的本质区别是：Polymarket 的价格波动和现实事件强相关。

我们可以在 K 线图上叠加事件时间线：
- 重要新闻发布时间点标记
- 相关人物发言时间点标记
- 官方结果公布倒计时

Verso 的 AI 新闻引擎是类似方向，但我们可以做得更直观（在图表上直接标记）。

**3. 跟单回测**

在决定跟踪某个地址之前，用户想知道"如果我过去一个月跟单这个地址，收益是多少"。

提供回测功能，输入地址 + 时间范围 + 跟单金额，输出模拟收益曲线。这是所有竞品都没有的功能。

**4. 功能整合**（新增）

参考 Hashdive 的整合度，但做得更好：
- K 线图 + 技术指标（对标 Hashdive）
- 鲸鱼追踪 + 跟单（对标 Bullpen）
- AI 新闻关联（借鉴 Verso）
- 社区化体验（借鉴 Betmoar 的 Discord 模式，做 Telegram 版）

**5. UMA 争议追踪**（新增借鉴点）

从 Betmoar 学习，增加 UMA 争议市场分析：
- 争议市场发现
- 投票进度追踪
- 潜在翻转机会提醒

这对于在结算前发现套利机会很有价值。

### 3.5 技术选型建议

**数据层**
- 实时数据：Polymarket 官方 WebSocket（免费）
- K线数据：官方 REST API `/v1beta1/report/trades/stats`
- 补充数据：PredictAPI（$29/月 Pro 版，更低延迟）

**前端**
- Web：React + TradingView Charting Library
- 图表：Lightweight Charts（TradingView 开源版）或 ECharts
- 移动端：Telegram Bot（MVP阶段），后续考虑 React Native

**后端**
- 数据聚合服务：Node.js/Go
- 地址分析：Python（ML 相关）
- 数据库：PostgreSQL（交易数据）+ Redis（实时缓存）
